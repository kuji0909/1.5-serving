# 1.5인분 — 온보딩 기능 명세서
> 작성일: 2026-04-24 | 버전: v1.0 | 대상 퍼소나: 이지수(P1), 박재현(P2)

---

## 1. 기능 개요 (Why)

### 해결하는 문제
- **P1 이지수**: 기존 나눔 앱(당근 등)이 식재료 전용 카테고리가 없어 올려도 반응이 없고, 등록 과정이 번거로워 이탈함 → *"올리는 과정이 복잡하지 않은 앱을 원한다."* (U1)
- **P2 박재현**: 첫 경험이 나쁘면 즉시 삭제하는 헤비유저. 위생 불신이 강해 첫 화면에서 "이건 믿을 수 있는 앱이다"라는 확신을 주지 못하면 이탈 → *"남의 식재료는 위생이 찝찝해 나눔 앱을 사용하지 않겠다."* (U10)

### 전략 연결
- **전략 1** (초근거리 비대면): 온보딩에서 거주 위치·선호 반경을 설정해 첫 피드부터 100m~도보 5분 매물만 노출
- **전략 2** (식품 신뢰 시스템): 품목 유형 분류(미개봉/신선/밀봉)를 온보딩 단계에서 유저에게 인지시켜 위생 불신 선제 해소

### 성공 가정
온보딩 완료 시 유저가 "이 앱은 내 동네 식재료 전용이구나, 위생도 관리되는구나"를 체감하고 첫 나눔 피드까지 도달한다.

## 2. 기능 목록 (What)

| 기능ID | 기능명 | 우선순위 | 대상 퍼소나 | 한 줄 설명 |
|--------|--------|----------|------------|-----------|
| F-01 | 회원가입 / 소셜 로그인 | P0 필수 | 공통 | 카카오·애플 로그인으로 3탭 이내 가입 완료 |
| F-02 | 거주 위치 설정 | P0 필수 | 공통 | GPS 자동 감지 or 수동 입력, 인증 방식 선택 |
| F-03 | 나눔 반경 선택 | P0 필수 | 공통 | 100m / 도보 5분 / 도보 10분 중 선택 |
| F-04 | 위생 신뢰 가이드 노출 | P1 중요 | P2 중심 | 품목 유형 분류 기준(미개봉·신선·밀봉)을 한 장으로 소개 |
| F-05 | 첫 피드 진입 | P0 필수 | 공통 | 설정한 반경 내 나눔 목록으로 바로 연결 |
| F-06 | 픽업 방식 선호 설정 | P1 중요 | P2 중심 | "직접 나감 / 상대방이 와야 함 / 상관없음" 선택 |
| F-07 | 푸시 알림 동의 | P1 중요 | 공통 | 반경 내 신규 나눔 등록 시 알림 수신 여부 |
| F-08 | 스킵 허용 | P2 선택 | 공통 | F-04·F-06·F-07은 스킵 가능, 나중에 설정에서 변경 가능 |

> **우선순위 기준**: P0 = 없으면 서비스 불가 / P1 = 핵심 가치 전달 / P2 = 경험 향상

## 3. 기능 상세 (How)

---

### F-01 회원가입 / 소셜 로그인

**트리거 조건**
앱 첫 실행 시 자동 진입. 기존 계정 없을 때.

**입력**
- 카카오 / 애플 OAuth 토큰
- (선택) 닉네임 — 자동 생성 기본값 제공

**처리 로직**
1. 소셜 로그인 버튼 탭
2. OAuth 인증 완료
3. 신규 유저 → 온보딩 F-02로 이동 / 기존 유저 → 피드로 바로 이동

**출력**
로그인 완료 토스트 메시지 + F-02 위치 설정 화면으로 자동 전환

**엣지 케이스**
- 네트워크 오류 → "잠시 후 다시 시도해주세요" 인라인 에러
- 소셜 계정 연동 거부 → 재시도 버튼 노출, 앱 종료 유도 없이 유지
- 동일 이메일로 다른 소셜 로그인 시도 → "이미 카카오로 가입된 계정입니다" 안내

---

### F-02 거주 위치 설정

**트리거 조건**
F-01 완료 직후 자동 진입.

**입력**
- GPS 위치 권한 허용 또는 주소 직접 입력

**처리 로직**
1. "내 위치 자동 설정" 버튼 → 위치 권한 요청 팝업
2. 허용 시: GPS 좌표 → 행정동 단위로 변환해 표시 ("서울 마포구 연남동")
3. 거부 시: 주소 검색 입력창 노출 (도로명 검색)
4. 확인 버튼 탭 → F-03으로 이동

**출력**
"OO동에서 나눔을 시작할게요!" 확인 문구 + 지도 미리보기 (선택)

**엣지 케이스**
- GPS 신호 약할 때 → 수동 입력 옵션 바로 노출
- 위치 권한 거부 후 주소 입력도 스킵 시 → 앱 재실행 시 재요청 (강제 아님)

---

### F-03 나눔 반경 선택

**트리거 조건**
F-02 완료 직후.

**입력**
3가지 옵션 중 1개 탭 선택
- 🟢 100m 이내 (우리 집 앞)
- 🟡 도보 5분 이내
- 🔵 도보 10분 이내

**처리 로직**
1. 옵션 탭 → 지도 위 반경 원이 실시간으로 변경
2. 해당 반경 내 현재 나눔 건수 미리보기 표시 ("현재 이 반경엔 나눔 3건이 있어요")
3. 확인 → F-04 or F-05 이동

**출력**
선택된 반경이 프로필에 저장, 피드 기본 필터로 적용

**엣지 케이스**
- 선택한 반경 내 나눔 0건 → "아직 이 반경엔 나눔이 없어요. 더 넓게 볼까요?" 제안 (강제 변경 아님)

---

### F-04 위생 신뢰 가이드

**트리거 조건**
F-03 완료 직후 자동 노출 (P2 유형 이탈 방지 핵심 장치).

**입력**
없음 (읽기 전용 화면). 하단 "확인했어요" 버튼 or "스킵" 텍스트 링크.

**처리 로직**
품목 유형 3가지를 카드 형식으로 소개:
- 🔒 **미개봉 포장** — 공장 밀봉 상태, 신뢰도 최고
- 🥬 **신선 식재료** — 유통기한·사진 첨부 필수
- 🥫 **밀봉 가공식품** — 통조림·세트 등 장기 보관 가능

**출력**
유저가 이 화면을 본 여부를 서버에 기록 (향후 신뢰 가이드 재노출 여부 판단용)

**엣지 케이스**
- 스킵 시 → 첫 나눔 상세 진입 시 툴팁으로 재노출

---

### F-05 첫 피드 진입

**트리거 조건**
온보딩 마지막 단계 완료 후 자동 이동.

**입력**
없음 (설정된 위치·반경 기반 자동 쿼리).

**처리 로직**
1. 설정 반경 내 나눔 목록 최신순 로드
2. 건수 0일 경우 → 빈 피드 대신 "첫 나눔을 올려보세요" CTA 노출
3. 첫 방문 유저에게 "나눔 올리기" 버튼 강조 (코치마크)

**출력**
나눔 카드 리스트 (품목 유형 뱃지 + 거리 + 등록 시간 표시)

---

### F-06 픽업 방식 선호 설정

**트리거 조건**
F-04 완료 후 또는 설정 탭에서 언제든 변경 가능.

**입력**
3가지 옵션:
- "내가 나가서 픽업할게요"
- "상대방이 내 위치로 와줬으면 해요"
- "상관없어요"

**처리 로직**
선택값 저장 → 피드에서 나눔 카드 필터링 기준으로 활용 (매칭 우선순위에 반영)

**엣지 케이스**
- 스킵 시 기본값 "상관없어요"로 저장

## 4. 화면 플로우 (Flow)

```
[스플래시]
  ↓ (자동)
[소셜 로그인 화면] ← F-01
  ↓ (로그인 완료)
[위치 설정] ← F-02
  ↓ (위치 확인)
[반경 선택] ← F-03  ⚠️ 이탈 위험①
  ↓ (반경 선택)
[위생 신뢰 가이드] ← F-04  ⚠️ 이탈 위험②
  ↓ (확인 or 스킵)
[픽업 방식 선호] ← F-06
  ↓ (선택 or 스킵)
[푸시 알림 동의] ← F-07
  ↓ (동의 or 거부)
[첫 피드] ← F-05  ✅ 온보딩 완료
```

**이탈 위험 포인트 & 대응**

| 포인트 | 이탈 원인 | 대응 방안 |
|--------|----------|----------|
| ⚠️ 위험① 반경 선택 | 반경 내 나눔 0건 → "이 앱 별거 없네" 판단 | 현재 건수 실시간 표시 + 넓은 반경 유도 제안 |
| ⚠️ 위험② 위생 가이드 | 화면 하나 더 보기 귀찮아 이탈 | 스킵 허용 + 텍스트 최소화 (카드 3장, 5초 내 읽기) |
| 위치 권한 거부 | 권한 팝업에서 거부 후 방법 몰라 이탈 | 권한 거부 시 즉시 "주소 직접 입력" 옵션 노출 |

## 5. 수락 조건 (Done Criteria)

### 기능 완료 기준
- [ ] 소셜 로그인(카카오/애플) 완료 후 3초 이내 위치 설정 화면 진입
- [ ] 위치 설정 → 반경 선택 → 피드 진입까지 최대 5단계, 60초 이내 완료 가능
- [ ] 반경 내 나눔 0건일 때 빈 피드가 아닌 대체 UI 노출
- [ ] 모든 P1 기능(F-04, F-06, F-07)은 스킵 가능하고 설정에서 재설정 가능
- [ ] 온보딩 완료 여부가 서버에 기록되어 재설치 시 재진행하지 않음

### 품질 기준
- [ ] 피드 첫 로딩 응답 시간 2초 이내 (3G 환경 기준)
- [ ] 온보딩 각 화면 전환 애니메이션 300ms 이내
- [ ] iOS / Android 양 플랫폼 동일 UX 보장

### 측정 지표 (KPI)

| 지표명 | 목표값 | 측정 방법 |
|--------|--------|----------|
| 온보딩 완료율 | 설치 후 60% 이상 피드 진입 | 퍼널 분석 (설치 → 피드 도달) |
| 위생 가이드 열람율 | 노출 대비 70% 이상 확인 클릭 | 이벤트 로그 |
| 반경 설정 완료율 | 90% 이상 | 단계별 드롭 분석 |
| D1 리텐션 | 온보딩 완료 유저 기준 40% 이상 | 코호트 분석 |
